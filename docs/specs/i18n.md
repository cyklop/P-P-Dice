# Feature Specification: Internationalisierung (i18n)

## Übersicht
Das System soll mehrsprachig werden. Deutsch bleibt die Standardsprache, Englisch wird als zweite Sprache hinzugefügt. Die Sprachauswahl erfolgt über URL-Präfixe (`/de/`, `/en/`) und ist per Browser-Locale Auto-Detection und manuellem Umschalter im Footer steuerbar.

## Ziele
- Alle UI-Texte über Übersetzungsdateien (JSON) statt Hardcoded-Strings
- Deutsch (Standard) + Englisch als unterstützte Sprachen
- SEO-freundliche URLs mit Sprachpräfix
- Nahtloses Umschalten ohne Datenverlust (Raum-State bleibt erhalten)

## Scope

### Inkludiert
- next-intl Integration mit Next.js App Router
- URL-Prefix Routing: `/de/room/ABC123`, `/en/room/ABC123`
- Auto-Detection via `Accept-Language` Header (bei Root `/`)
- Manueller Language Switcher: Globe-Icon im Footer
- Übersetzungsdateien für DE + EN (alle UI-Strings)
- Middleware für Locale-Redirect

### Exkludiert
- Serverseitige Logs/Console-Ausgaben (bleiben Englisch)
- Spielernamen (User-Input, nicht übersetzt)
- Weitere Sprachen (Architektur unterstützt es, aber nicht im Scope)

## Technische Anforderungen

### Library: next-intl
- Optimal für Next.js 15 App Router
- Server Components + Client Components Support
- URL-basiertes Routing mit Middleware

### Verzeichnisstruktur
```
src/
  i18n/
    request.ts          # next-intl Konfiguration
    routing.ts          # Routing-Konfiguration (locales, defaultLocale)
  messages/
    de.json             # Deutsche Übersetzungen
    en.json             # Englische Übersetzungen
  app/
    [locale]/           # Dynamic locale segment
      layout.tsx        # NextIntlClientProvider wrapper
      page.tsx          # Landing page
      room/
        [code]/
          page.tsx      # Room page
  middleware.ts         # Locale detection + redirect
```

### Routing
- `defaultLocale: 'de'`
- `locales: ['de', 'en']`
- Root `/` → Redirect zu `/de/` oder `/en/` basierend auf Browser-Locale
- Alle bestehenden Routen bekommen `[locale]` Prefix

### Übersetzungsschlüssel-Konvention
- Namespace-basiert: `common`, `room`, `lobby`, `dice`, `stats`
- Flache Keys: `room.playerList`, `room.kickPlayer`, `dice.rollButton`

## UI/UX Anforderungen

### Language Switcher
- Position: Footer, rechte Seite
- Design: Globe-Icon (🌐 oder Heroicon) + aktuelle Sprache (DE/EN)
- Klick öffnet Dropdown mit verfügbaren Sprachen
- Wechsel navigiert zur gleichen Seite mit neuem Locale-Prefix
- Kein Datenverlust: Socket-Verbindung bleibt bestehen (reconnect via Token)

### Auto-Detection
- Nur bei erstmaligem Besuch (Root `/`)
- Danach: URL-Locale hat Priorität
- Kein Cookie-basiertes Locale-Tracking nötig (URL ist die Source of Truth)

## Edge Cases & Error Handling
- **Ungültiger Locale in URL**: Middleware redirected zu Default (`/de/`)
- **Fehlender Übersetzungsschlüssel**: Fallback zu Deutsch, Warning in Console
- **Raum-Wechsel der Sprache**: Socket reconnect via Token, State bleibt erhalten
- **Geteilte Room-Links**: Link enthält Locale, empfangender User sieht diese Sprache (kann wechseln)
